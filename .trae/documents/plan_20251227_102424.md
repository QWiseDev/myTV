# Upstash 调用优化方案

## 目标

减少 Upstash Redis 调用次数以降低按量付费费用

## 优化措施

### 1. 删除**访问日志 (最高优先级)**

* 删除`upstash.db.ts` 中的 `saveAccessLog()` 方法

### 2. **优化播放进度保存策略**

* 延长 Upstash 模式下的保存间隔 (30s)

### 3. **缓存播放统计结果**

* 为 `getUserPlayStat()` 添加本地内存缓存 (TTL 60s)

* 避免短时间内重复获取相同统计数据

* **预期减少**: 高频访问时的重复查询

### 4. **合并登录统计操作为单个 INCR 命令**

* 当前: `GET` + `SET` (2次调用)

* 优化: 使用 `INCRBY` + `HSET` (1次调用或使用管道)

* **预期减少**: 每次登录减少 1 次调用

### 5. **优化缓存统计页面**

* 限制 `KEYS cache:*` 的执行频率

* 添加缓存键数量统计的定时任务，而非实时查询

* **预期减少**: 管理员页面的全库扫描

<br />

## 实施顺序

1. ✅ 禁用访问日志 (最快见效)
2. ✅ 延长播放进度保存间隔
3. ✅ 添加统计查询缓存
4. ✅ 优化登录统计操作
