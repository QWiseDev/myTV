# 老虎机后台抽奖系统

## 功能概述

已成功实现基于用户系统的老虎机后台抽奖功能，包括完整的用户管理、金币系统和抽奖逻辑。

## 已实现的功能

### 1. 后端API接口

#### `/api/slot/spin` - 老虎机抽奖接口
- **POST**: 进行一次老虎机抽奖
- **GET**: 获取当前用户数据

**请求格式**:
```typescript
// POST请求无需参数，使用认证Cookie
// 请求会自动扣除10金币作为投注金额
```

**响应格式**:
```typescript
{
  success: true,
  results: ["j", "n", "t", "m"], // 4个符号结果
  result: {
    type: "win" | "punishment" | "none",
    name: "姬霓太美",
    multiplier: 128,
    winAmount: 1280, // 赢取金额
    betAmount: 0, // 扣除金额（惩罚时使用）
  },
  user: {
    coins: 1000, // 当前金币
    totalSpins: 10, // 总抽奖次数
    totalWins: 3, // 总获胜次数
    biggestWin: 1280, // 最大单次赢取
    isBanned: false, // 是否被封禁
    banEndTime: 0, // 封禁结束时间
  }
}
```

#### `/api/slot/user` - 用户管理接口
- **GET**: 获取用户详细数据
- **DELETE**: 重置用户账户
- **PATCH**: 修改用户金币（管理员功能）

### 2. 用户系统

#### 初始配置
- **初始金币**: 1000枚
- **每次投注**: 10枚金币
- **数据存储**: 使用项目现有的缓存系统（30天过期）

#### 用户数据结构
```typescript
interface SlotUserData {
  coins: number; // 用户金币
  totalSpins: number; // 总抽奖次数
  totalWins: number; // 总获胜次数
  biggestWin: number; // 最大单次赢取
  isBanned: boolean; // 是否被封禁
  banEndTime: number; // 封禁结束时间
  lastSpinTime: number; // 上次抽奖时间
  initialCoins: number; // 初始金币
  createdTime: number; // 账户创建时间
}
```

### 3. 游戏规则

#### 符号权重系统（总计980）
- 姬 (j): 140权重 - 10金币价值
- 霓 (n): 140权重 - 15金币价值
- 太 (t): 140权重 - 20金币价值
- 美 (m): 140权重 - 25金币价值
- 老 (lq): 100权重 - 30金币价值
- 八 (zft): 50权重 - 40金币价值
- 蛋 (bdk): 30权重 - 50金币价值

#### 中奖组合
**大奖组合**:
- 姬霓太美 (j,n,t,m): 128倍奖励
- 姬霓霓霓 (j,n,n,n): 64倍奖励
- 霓霓霓霓 (n,n,n,n): 32倍奖励
- 姬太太太 (j,t,t,t): 16倍奖励
- 老老 (lq,lq): 8倍奖励
- 八八 (zft,zft): 4倍奖励
- 蛋蛋 (bdk,bdk): 4倍奖励

**惩罚组合**:
- 1个律师函 (lq): 扣除1倍投注
- 3个律师函: 扣除3倍投注 + 封禁2分钟
- 1个黑蛋 (bdk): 扣除2倍投注
- 2个黑蛋: 扣除5倍投注
- 3个黑蛋: 扣除10倍投注 + 封禁5分钟
- 1个八卦 (zft): 扣除2倍投注
- 2个八卦: 扣除4倍投注

### 4. 前端功能

#### 用户管理面板
- **位置**: 金币显示旁的用户图标按钮
- **功能**:
  - 显示用户统计数据（金币、获胜次数、最大赢取、历史记录）
  - 显示账户状态（封禁信息）
  - 重置账户功能（可选择500/1000/5000初始金币）

#### 自动数据同步
- 组件加载时自动获取用户数据
- 每次抽奖后同步更新用户状态
- 支持封禁状态自动检测和解除

## 安全特性

### 1. 用户认证
- 使用项目现有的Cookie认证系统
- 所有API调用都需要有效的用户登录
- 防止未授权访问

### 2. 数据验证
- 严格的输入参数验证
- 金币数量范围限制（0-100000）
- 单次操作金额限制（最大10000金币）

### 3. 防作弊机制
- 抽奖结果完全由后端生成
- 权重随机算法确保公平性
- 封禁系统防止恶意行为

## 技术实现

### 1. 后端架构
- 基于Next.js API Routes
- 集成现有数据库抽象层
- 使用缓存系统存储用户数据

### 2. 前端集成
- React Hooks状态管理
- Framer Motion动画效果
- 响应式设计支持

### 3. 数据持久化
- 用户数据缓存存储（30天过期）
- 支持项目现有的多种存储后端
- 自动数据同步机制

## 使用方法

### 1. 开始游戏
1. 用户登录系统
2. 打开老虎机页面
3. 点击旋转按钮进行抽奖

### 2. 管理账户
1. 点击金币旁的用户图标
2. 查看个人统计数据
3. 可选择重置账户并重新开始

### 3. API调用示例
```typescript
// 进行抽奖
const response = await fetch('/api/slot/spin', {
  method: 'POST',
});
const data = await response.json();

// 获取用户数据
const userResponse = await fetch('/api/slot/spin');
const userData = await userResponse.json();

// 重置账户
const resetResponse = await fetch('/api/slot/user', {
  method: 'DELETE',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ initialCoins: 1000 }),
});
```
